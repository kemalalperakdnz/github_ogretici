GitHub ile Kod Yönetimi

GitHub, yazılım geliştirme ve kod yönetimi için web tabanlı bir platformdur. Geliştiricilere kodlarını depolamak, değişiklikleri izlemek, projelerini yönetmek ve işbirliği yapmak için güçlü araçlar sunar. Açık kaynak projelerin merkezi haline gelmiş bu platform, modern yazılım geliştirme süreçlerinin vazgeçilmez bir parçasıdır.

GitHub Nedir?

GitHub'ın Tanımı ve Tarihi

GitHub, 2008 yılında Tom Preston-Werner, Chris Wanstrath ve PJ Hyett tarafından kurulmuş, Git versiyon kontrol sistemini temel alan bir kod barındırma platformudur. 2018 yılında Microsoft tarafından 7.5 milyar dolara satın alınmıştır.

Neden GitHub Kullanmalıyız?

Versiyon Geçmişi

GitHub, projenizin tüm geçmişini saklar. Her değişiklik, kim tarafından ne zaman yapıldığı bilgisiyle birlikte kaydedilir. Bu sayede herhangi bir zamandaki projenizin durumuna geri dönebilirsiniz.

İşbirliği

Birden fazla geliştirici aynı projede çalışırken, GitHub çakışmaları (conflict) yönetmeyi ve değişiklikleri birleştirmeyi kolaylaştırır. Ekip üyeleri aynı anda farklı özellikler üzerinde çalışabilir.

Dallanma Kolaylığı

GitHub'ın güçlü dallanma mekanizması, yeni özellikleri ana koddan izole bir şekilde geliştirmenize olanak tanır. Bu, stabil bir ana kod tabanını korurken yeni özellikler eklemeyi mümkün kılar.

Merkezi Depolama

Kodunuz güvenli bir şekilde bulutta saklanır, böylece yerel bilgisayarınızda veri kaybı yaşasanız bile projeniz güvende kalır.

Veri Bütünlüğü

GitHub, her commit'in benzersiz bir hash değeriyle tanımlanmasını sağlayarak, projenizin bütünlüğünü korur. Bu, herhangi bir değişikliğin fark edilmeden yapılmasını imkansız hale getirir.

Performans

GitHub, büyük projeler ve binlerce dal (branch) ile bile hızlı çalışacak şekilde tasarlanmıştır. Çoğu işlem yerel olarak gerçekleştiği için ağ gecikmesi yaşanmaz.

GitHub ile Başlarken

GitHub'da yolculuğunuza başlamak için izlemeniz gereken adımlar aşağıdadır. Bu basit adımları takip ederek kısa sürede projelerinizi GitHub'da yönetmeye başlayabilirsiniz.

1

GitHub Hesabı Oluşturun

github.com adresine gidin ve yeni bir hesap oluşturun. E-posta adresinizi doğrulayın ve güçlü bir şifre belirleyin.

Hesap Oluştur
2

GitHub Desktop'u Yükleyin

GitHub'la etkileşime geçmek için GitHub Desktop uygulamamasını indirip yükleyin. Bu kullanıcı dostu arayüz, komut satırı kullanmadan GitHub'la çalışmanızı sağlar.

GitHub Desktop İndir
3

İlk Repository'nizi Oluşturun

GitHub hesabınıza giriş yapın ve sağ üst köşedeki + simgesine tıklayarak "New repository" seçeneğini seçin. Repository'nize bir isim verin ve istediğiniz ayarları yapın.

Repository Oluşturma 
1. Repository adını girin ("my-first-repo")
2. Kısa bir açıklama ekleyin
3. Public veya Private seçeneğini belirleyin
4. "Initialize with a README" seçeneğini işaretleyin
5. "Create repository" butonuna tıklayın
4

Repository'nizi Klonlayın

GitHub Desktop uygulamasını açın ve oluşturduğunuz repository'yi yerel bilgisayarınıza klonlayın. Bu işlem, repository'nin bir kopyasını bilgisayarınıza indirecektir.

GitHub Desktop ile Klonlama 
1. GitHub Desktop'ta "File > Clone repository" seçeneğine tıklayın
2. GitHub.com sekmesinden repository'nizi seçin
3. Yerel bir klasör belirleyin
4. "Clone" butonuna tıklayın
5

Değişiklikler Yapın ve Commit Edin

Yerel klasörünüzde dosyalarınızı düzenleyin veya yeni dosyalar ekleyin. Sonra GitHub Desktop'a dönün, değişikliklerinizi gözden geçirin ve bir commit mesajı yazarak kaydedin.

Commit ve Push İşlemi 
1. Değişikliklerinizi gözden geçirin
2. Commit mesajını yazın ("Add new features")
3. "Commit to main" butonuna tıklayın
4. "Push origin" butonuna tıklayarak değişikliklerinizi GitHub'a gönderin

Commit mesajlarınızı açıklayıcı ve anlaşılır yazın. Bu, projenizin geçmişini takip etmeyi kolaylaştırır.

Hazır mısınız?

Bu adımları tamamladıktan sonra, GitHub'da projelerinizi yönetmeye başlayabilirsiniz. Şimdi GitHub'la kod yolculuğunuza başlayın!

GitHub'a Kayıt Ol

GitHub'da Kod Yönetimi

Repository Oluşturma

GitHub'da yeni bir repository (depo) oluşturmak, projelerinizi yönetmenin ilk adımıdır. GitHub web arayüzünden kolayca yeni bir repository oluşturabilirsiniz.

1. GitHub'da sağ üst köşedeki + simgesine tıklayın
2. "New repository" seçeneğini seçin
3. Repository adı ve açıklaması girin
4. Visibility seçeneğini belirleyin (Public/Private)
5. "Create repository" butonuna tıklayın

Repository Klonlama

GitHub'daki bir repository'yi yerel bilgisayarınıza kopyalamak için klonlama işlemi yapabilirsiniz.

$ git clone https://github.com/kullanici/depo.git
Cloning into 'depo'...

Değişiklikleri Hazırlama

Dosyalarda yaptığınız değişiklikleri GitHub'a göndermek için önce hazırlama alanına eklemeniz gerekir.

$ git add dosya.txt
$ git add .  # Tüm değişiklikleri ekler

Değişiklikleri Kaydetme

Hazırlama alanındaki değişiklikleri bir commit ile kaydetmeniz gerekir. Her commit, projenizin bir anlık görüntüsünü oluşturur.

$ git commit -m "İlk commit"
[main f7d2e3a] İlk commit
1 file changed, 10 insertions(+)

Durum Kontrolü

Çalışma dizininizdeki değişikliklerin durumunu kontrol etmek için status komutunu kullanabilirsiniz.

$ git status
On branch main
Changes to be committed:
  modified: index.html
Changes not staged for commit:
  modified: style.css

GitHub'a Yükleme

Yerel değişikliklerinizi GitHub'daki repository'nize göndermek için push komutunu kullanabilirsiniz.

$ git push origin main
Enumerating objects: 5, done.
Counting objects: 100% (5/5), done.
Writing objects: 100% (3/3), 353 bytes, done.
To https://github.com/kullanici/depo.git
  a1b2c3d..f7d2e3a  main -> main

GitHub'dan Güncelleme Alma

GitHub'daki repository'nizde yapılan değişiklikleri yerel kopyanıza almak için pull komutunu kullanabilirsiniz.

$ git pull origin main
From https://github.com/kullanici/depo
 * branch            main       -> FETCH_HEAD
Updating a1b2c3d..e5f6g7h
Fast-forward
 README.md | 5 +++++
 1 file changed, 5 insertions(+)

GitHub İş Akışı ve İşbirliği

1. Değişiklikler Yapın

Projenizde gerekli değişiklikleri yapın. Yeni dosyalar ekleyin, mevcut dosyaları düzenleyin veya silin.

2. Değişiklikleri Ekleyin

Yaptığınız değişiklikleri hazırlama alanına ekleyin:

git add .

3. Commit Edin

Değişikliklerinizi anlamlı bir mesajla commit edin:

git commit -m "Açıklayıcı commit mesajı"

4. Push Edin

Değişikliklerinizi GitHub'daki uzak depoya gönderin:

git push origin main

5. Pull Request Oluşturun

Değişikliklerinizi ana projeye birleştirmek için GitHub üzerinden bir Pull Request oluşturun. Bu, kodunuzun incelenmesini ve tartışılmasını sağlar.

GitHub'da Verimlilik İpuçları

Anlamlı Commit Mesajları Yazın

Her commit mesajı, yapılan değişikliği açıkça anlatmalıdır. "Düzeltme" yerine "Navigasyon menüsündeki bozuk bağlantı düzeltildi" gibi açıklayıcı mesajlar kullanın.

Sık Sık Commit Yapın

Büyük değişiklikler yerine, küçük ve anlamlı değişikliklerle sık sık commit yapın. Bu, hata ayıklamayı kolaylaştırır ve geçmişi daha anlaşılır hale getirir.

.gitignore Dosyası Kullanın

GitHub repository'nize eklenmemesi gereken dosyaları belirlemek için .gitignore dosyası oluşturun. GitHub, farklı programlama dilleri için hazır şablonlar sunar.

Sürümleri Etiketleyin

Önemli dağıtımlar için git tag komutunu kullanarak sürüm etiketleri oluşturun. Bu, belirli bir sürüme kolayca erişmenizi sağlar.

GitHub Flow'u Benimseyin

GitHub'ın önerdiği iş akışı modelini (GitHub Flow) benimseyin. Bu model, branch'ler, pull request'ler ve merge işlemleri için basit ve etkili bir yaklaşım sunar.

GitHub Actions Kullanın

GitHub Actions ile CI/CD süreçlerinizi otomatikleştirin. Kod testleri, derleme ve dağıtım işlemlerini otomatik olarak gerçekleştirerek geliştirme sürecinizi hızlandırın.
